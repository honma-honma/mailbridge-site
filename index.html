PROBLEM

毎日のメール、こんな手間に追われていませんか？

確認・仕分け・入力に時間が消える

受信したメールを読み、振り分け、返信し、受注台帳に転記。この一連の作業だけで毎日まとまった時間が奪われています。

重要メールの見落とし・対応漏れ

大量のメールに埋もれ、急ぎの問い合わせや見積依頼への返信が遅れてしまう。気づかぬうちに信頼を損ねるリスクがあります。

担当者しか分からない属人化

「あの案件の状況はあの人しか知らない」。担当者が不在になると業務が止まり、引き継ぎにも苦労します。

MailBridgeは「AIが下ごしらえ → 人は確認するだけ」で、この手間を圧縮します。

HOW IT WORKS

導入は、かんたん3ステップ

今お使いのメール環境はそのまま。難しい準備は要りません。

01

ダウンロードしてインストール

Windows用インストーラをダウンロードして実行するだけ。Python等の事前準備は不要です。

02

メール接続＆取引先を登録

Outlook / IMAPのメールアカウントを接続し、処理したい取引先を「顧客マスタ」に登録します。

03

「メールを処理」を押すだけ

AIが仕分け・下書き・台帳入力まで下ごしらえ。あとは内容を確認して送るだけです。

FEATURES

現場の手間を減らす、7つの機能

「下ごしらえはAI、最終判断は人」。安全を最優先に設計しています。

AI自動仕分け

受信メールを「スルー／自動返信／要確認」に振り分け。迷ったら必ず「要確認」に寄せて、誤送信を防ぐ安全設計です。

返信下書きの自動生成

AIが返信の下書きを作成。人が確認・修正して送信します。署名も自動で付与されます。

受注台帳へ自動入力

条件を満たす確実なメールのみ、Googleスプレッド/Excelへ自動記帳。「要確認」はスキップします。

社内チャット連携

要確認メールを「案件カード」として社内に通知・共有。担当者以外も状況を把握でき、属人化を防ぎます。

顧客マスタで対象を限定

登録した取引先からのメールだけを処理。無関係なメールには一切触りません。

Outlook / IMAP どちらでも

今のメール環境のまま導入できます。メールアドレスを変える必要はありません。

マルチAI対応（BYOK）

Gemini / OpenAI / Claude / Ollama を切替可能。AIキーは会社で1つ設定すれば、全メンバーに自動で適用されます。

SECURITY & TRUST

大切なメールだから、安心して使える設計

情報の取り扱いとAIの限界について、正直にお伝えします。

メール本文はサーバーに保存しない

サーバーに記録するのは判定理由・要約のみ。メール本文そのものは保存しません。

パスワード・AIキーは端末内に暗号化保存

メールのパスワードやAIキーは端末内で暗号化して保管し、原則サーバーには送りません。

会社ごとにデータを分離

マルチテナント・行レベルセキュリティにより、他社とデータが混ざることはありません。

AIは万能ではありません（正直に）

AIの分類・返信には誤りが含まれる可能性があります。重要なメールは必ず人が確認する運用を推奨します。
